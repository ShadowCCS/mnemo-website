Contribute Code

Last updated: July 24, 2025 7 min read

Want to contribute to mnemo's core development? This guide will walk you through the process of setting up your development environment and making your first contribution.

Getting Started

Before you start contributing, make sure you have:

  • A GitHub account
  • Visual Studio 2022 or later (recommended IDE)
  • .NET 8.0 SDK or later
  • Basic knowledge of C# and XAML

Development Environment Setup

Follow these steps to set up your development environment:

  1. Fork the Repository
    • Visit the mnemo GitHub repository
    • Click the "Fork" button in the top-right corner
    • Clone your fork locally: git clone https://github.com/YOUR-USERNAME/mnemo.git
  2. Install Dependencies
    • Navigate to the project directory
    • Run dotnet restore to install dependencies
    • Install required Visual Studio workloads if prompted
  3. Configure Development Settings
    • Copy appsettings.example.json to appsettings.Development.json
    • Update configuration values as needed
Note: Make sure to read our contribution guidelines before making any changes.

Project Structure

The mnemo codebase is organized into several key projects:

  • Mnemo.Core - Core business logic and domain models
  • Mnemo.UI - Avalonia UI implementation
  • Mnemo.Data - Data access and storage
  • Mnemo.Extensions - Extension system implementation
  • Mnemo.Games - Game engine integration
  • Mnemo.Tests - Unit and integration tests

Making Changes

Follow these steps when making changes to the codebase:

  1. Create a Branch
    • Create a new branch for your feature: git checkout -b feature/your-feature-name
    • For bug fixes: git checkout -b fix/bug-description
  2. Write Code
    • Follow our coding standards and conventions
    • Add XML documentation for public APIs
    • Include unit tests for new features
  3. Test Your Changes
    • Run unit tests: dotnet test
    • Perform manual testing
    • Check for any UI/UX inconsistencies
  4. Submit Your Changes
    • Commit your changes with a descriptive message
    • Push to your fork: git push origin feature/your-feature-name
    • Create a pull request on GitHub
Pro Tip: Use conventional commits format for your commit messages (e.g., "feat: add dark mode support" or "fix: resolve memory leak in timer").

Code Review Process

After submitting your pull request:

  1. We will review your code and give you feedback
  2. Address any feedback or requested changes
  3. Once approved, your code will be merged into the main branch

Best Practices

Follow these best practices when contributing:

  • Keep pull requests focused and small
  • Write clear commit messages
  • Follow SOLID principles
  • Follow the coding standards and conventions
  • Follow the MVVM pattern
  • Use the tools mnemo provides, such as using the overlay service, notification service, etc, instead of reinventing the wheel.

Getting Help

If you need help or have questions:

  • Join our Discord community
  • Check existing GitHub issues
  • Read our FAQ
  • Contact the core team
