Welcome to the developer portal! Here you can find all the information you need to start developing for mnemo.

Introduction

Developers are the backbone of the mnemo community. We are always looking for new developers to join us and help us make mnemo better, whatever you want to make, we want to help you.

This section of the documentation is for developers who want to contribute to the mnemo project. Here we will cover everything from how mnemo is built, how to contribute to the project and how to make games and extensions.

Development Opportunities

There are several ways you can contribute to mnemo:

  • Core Development - Help improve the main mnemo application by contributing code, fixing bugs, or implementing new features
  • Extension Development - Create new extensions to add functionality to mnemo
  • Game Development - Build educational games that integrate with mnemo's data
  • Documentation - Help improve our documentation and guides
  • Testing - Test new features and report bugs

Getting Started

Before you begin development, make sure you have:

  • Familiarized yourself with our contribution guidelines
  • Set up your development environment
  • Joined our developer community
Note: All development for mnemo should follow our coding standards and best practices to maintain consistency across the project.

Technical Overview

mnemo is built using modern web technologies and follows a modular architecture:

  • UI Framework: Avalonia C# Open Source UI framework
  • Pattern: MVVM
  • Database: SQLite
  • Extension System: Custom plugin architecture
  • Game Engine: Embedded MonoGame C# Open Source game engine
Pro Tip: Check out our sample projects to get a quick start with development.

Development Resources

  • SDK Downloads - Development kits for extensions and games
  • Code Examples - Sample code and templates
  • Developer Tools - Debugging and testing tools
  • Design Guidelines - UI/UX guidelines for consistent user experience
